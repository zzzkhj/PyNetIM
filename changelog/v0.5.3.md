# v0.5.3 更新日志

**发布日期**: 2026-04-16

---

## 概述

v0.5.3 是一个兼容性修复和代码重构版本，主要解决了跨平台编译问题，降低 C++ 标准要求以提高编译器兼容性，同时对深度学习模块进行了代码重构。

## 🐛 Bug 修复

### utils 模块参数签名显示问题

PyBind11 自动生成的函数签名格式不符合 Python 类型提示规范，导致文档显示不正确。

**问题**: `generate_rr_sets` 和 `sample_rr_set_ic` 函数的参数签名显示为 C++ 风格。

**修复**: 禁用 PyBind11 自动签名，在 docstring 中手动定义 Python 风格的类型提示。

**修改文件**:
- `src/pynetim/cpp/bindings/utils/rr_utils_bind.cpp`
- `src/pynetim/cpp/bindings/utils_bind.cpp`

**修改内容**:
```cpp
// 修复前
m.def("sample_rr_set_ic", &sample_rr_set_ic, "Sample a single RR set...");

// 修复后
{
    py::options options;
    options.disable_function_signatures();
    
    m.def("sample_rr_set_ic", &sample_rr_set_ic,
        R"doc(sample_rr_set_ic(graph: IMGraph, seed: int | None = None) -> list[int]
        
Sample a single Reverse Reachable (RR) set using IC model.
...)doc");
}
```

### macOS ARM64 编译失败

GitHub Actions 中 macOS ARM64 平台编译失败，原因是 `-mavx2` 编译选项不兼容 ARM 架构。

**问题**: AVX2 是 x86 特有的 SIMD 指令集，ARM 架构不支持。

**修复**: 在 `setup.py` 中添加平台检测，macOS ARM64 平台跳过 `-mavx2` 选项。

```python
# 修复前
extra_compile_args=['-O3', '-mavx2']

# 修复后
import platform
extra_compile_args = ['-O3']
if not (sys.platform == "darwin" and platform.machine() == "arm64"):
    extra_compile_args.append('-mavx2')
```

### 跨平台编译问题

修复了 GitHub Actions 中 Linux 和 macOS 平台的构建失败问题。

**问题原因**：
- `<format>` 是 C++20 特性，但 GCC < 13 版本不完全支持
- `-mavx2` 编译选项在 Windows MSVC 下不识别

**解决方案**：
- 使用 `std::ostringstream` 替代 `std::format`
- 根据平台自动选择正确的编译选项

### 编译器要求降低

| 平台 | 修复前 | 修复后 |
|------|--------|--------|
| GCC | 10+ | 8+ |
| Clang | 10+ | 7+ |
| MSVC | 19.28+ | 19.14+ |

## ✨ 改进

### C++ 标准从 C++20 降至 C++17

移除对 C++20 `<format>` 头文件的依赖，改用 C++17 兼容的 `std::ostringstream`，提高编译器兼容性。

**修改文件**：
- `src/pynetim/cpp/include/graph/Graph.h`

**修改内容**：
```cpp
// 修复前 (C++20)
#include <format>
throw std::runtime_error(std::format("Edge ({}, {}) does not exist", u, v));

// 修复后 (C++17)
#include <sstream>
std::ostringstream oss;
oss << "Edge (" << u << ", " << v << ") does not exist";
throw std::runtime_error(oss.str());
```

### 平台特定编译选项

`setup.py` 现在根据操作系统自动选择正确的编译选项：

```python
if sys.platform == "win32":
    extra_compile_args = ["/std:c++17", "/O2"]
else:
    extra_compile_args = ["-std=c++17", "-O3", "-mavx2"]
```

### 深度学习模块代码重构

#### 移除算法类冗余 train() 方法

算法类（如 `BiGDNAlgorithm`, `BiGDNSAlgorithm`）中原本包含 `train()` 方法，与独立的 `Trainer` 类功能重复。

**修复**: 移除算法类中的 `train()` 方法，训练功能统一由 `Trainer` 类提供。

| 类 | 移除方法 |
|---|---------|
| BiGDNAlgorithm | `train()` |
| BiGDNSAlgorithm | `train()` |

#### 移除算法类中的训练参数

算法类（如 `BiGDNAlgorithm`, `BiGDNSAlgorithm`）原本包含大量训练相关参数，但这些参数仅在训练阶段使用，推理阶段不需要。

**修复**: 从算法类 `__init__` 中移除训练参数，简化接口。

**移除的参数**:

| 类 | 移除的参数 |
|---|-----------|
| BiGDNAlgorithm | `gamma`, `epsilon`, `lr`, `target_update`, `n_steps`, `ntype` |
| BiGDNSAlgorithm | `gamma`, `epsilon`, `lr`, `target_update`, `n_steps`, `ntype`, `alpha`, `teacher_path`, `load_teacher` |

**保留的参数**（推理所需）:
- `graph` - 输入图对象
- `num_features` - 特征维度
- `device` - 计算设备
- `pretrained` - 是否使用预训练权重
- `weights_path` - 本地权重路径
- `encoder_path` - NodeEncoder 预训练权重路径（仅 BiGDN）
- `verbose` - 是否输出详细信息

**使用示例**:
```python
# 修复前
algo = BiGDNAlgorithm(
    graph, num_features=64,
    gamma=0.99, epsilon=1.0, lr=0.001,  # 训练参数
    target_update=100, n_steps=2, ntype='DQN',  # 训练参数
    device='auto', pretrained=True
)

# 修复后
algo = BiGDNAlgorithm(
    graph, num_features=64,
    device='auto', pretrained=True
)
```

#### 移除 save_weights() 方法

算法类中的 `save_weights()` 方法仅在训练后保存权重时使用，推理阶段不需要。

**修复**: 从 `BiGDNBaseAlgorithm` 中移除 `save_weights()` 方法。

**替代方案**: 训练时使用 `Trainer` 类保存权重。

#### Trainer 内部方法私有化

`Trainer` 类中的内部方法改为私有方法，添加下划线前缀，提高封装性。

| 类 | 方法变更 |
|---|---------|
| BiGDNTrainer | `select_action()` → `_select_action()` |
| BiGDNTrainer | `memorize()` → `_memorize()` |
| BiGDNTrainer | `fit()` → `_fit()` |
| BiGDNTrainer | `update_target()` → `_update_target()` |
| ToupleGDDTrainer | `select_action()` → `_select_action()` |
| ToupleGDDTrainer | `memorize()` → `_memorize()` |
| ToupleGDDTrainer | `fit()` → `_fit()` |
| ToupleGDDTrainer | `update_target()` → `_update_target()` |

#### 创建 DRL 算法基类

新增 `BaseDRLAlgorithm` 和 `BiGDNBaseAlgorithm` 基类，封装深度强化学习算法的公共逻辑。

**继承关系**:
```
BaseDLAlgorithm (深度学习基类)
    └── BaseDRLAlgorithm (深度强化学习基类)
            ├── ToupleGDDAlgorithm
            ├── S2VDQNAlgorithm
            └── BiGDNBaseAlgorithm (BiGDN 特有基类)
                    ├── BiGDNAlgorithm
                    └── BiGDNSAlgorithm
```

**公共方法**: `run()` 方法统一在基类中实现，子类只需实现 `_select_seeds()` 方法。

#### 统一 device 参数默认值

所有算法类的 `device` 参数默认值统一为 `'auto'`，使用基类的 `_get_device()` 方法处理。

| 类 | 修复前 | 修复后 |
|---|--------|--------|
| BiGDNAlgorithm | `device=None` | `device='auto'` |
| BiGDNSAlgorithm | `device=None` | `device='auto'` |

#### 重构 Trainer 参数

将训练配置参数从 `train()` 方法移到 `__init__` 中，减少方法参数数量，提高代码可读性。

**移动的参数**:
- `eps_start`, `eps_end`, `eps_decay_steps` - 探索率配置
- `pretrain_episodes`, `episodes_per_epoch` - 训练回合数
- `eval_interval`, `save_interval`, `target_update_interval` - 间隔设置
- `method`, `num_trials` - 影响力估计方法
- `verbose` - 日志显示

**使用示例**:
```python
# 修复前
trainer = BiGDNTrainer(num_features=64)
trainer.train(
    graphs, budget, num_epochs, save_path,
    eps_start=1.0, eps_end=0.05, eps_decay_steps=10000,
    pretrain_episodes=100, episodes_per_epoch=10,
    eval_interval=10, save_interval=100,
    target_update_interval=100, verbose=True
)

# 修复后
trainer = BiGDNTrainer(
    num_features=64,
    eps_start=1.0, eps_end=0.05, eps_decay_steps=10000,
    pretrain_episodes=100, episodes_per_epoch=10,
    eval_interval=10, save_interval=100,
    target_update_interval=100, verbose=True
)
trainer.train(graphs, budget, num_epochs, save_path)
```

#### 统一 n_steps 参数命名

ToupleGDD 模块中的 `n_step` 参数统一改为 `n_steps`，与 BiGDN 模块保持一致。

| 类 | 修复前 | 修复后 |
|---|--------|--------|
| ToupleGDDTrainer | `n_step` | `n_steps` |
| S2VDQNTrainer | `n_step` | `n_steps` |

## 升级指南

### 从 v0.5.2 升级

**注意**: 此版本包含 API 变更，如果您使用了 BiGDN/BiGDNS 算法类的训练参数，需要修改代码。

```bash
pip install --upgrade pynetim
```

**API 变更**:
- BiGDNAlgorithm 和 BiGDNSAlgorithm 移除了训练参数（gamma, epsilon, lr 等）
- 移除了 save_weights() 方法

**迁移指南**:
```python
# 旧代码（v0.5.2）
algo = BiGDNAlgorithm(
    graph, num_features=64,
    gamma=0.99, epsilon=1.0, lr=0.001,
    target_update=100, n_steps=2, ntype='DQN',
    device='auto', pretrained=True
)
algo.save_weights('weights.pth')

# 新代码（v0.5.3）
algo = BiGDNAlgorithm(
    graph, num_features=64,
    device='auto', pretrained=True
)
# save_weights 已移除，训练时使用 Trainer 保存权重
```
