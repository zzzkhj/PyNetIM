# v0.5.2 更新日志

**发布日期**: 2026-04-16

---

## 概述

v0.5.2 是一个重要更新，新增深度学习影响力最大化算法模块，支持 ToupleGDD、S2V-DQN 和 BiGDN 三种深度强化学习算法，并提供完整的训练框架。同时修复了 pybind 绑定代码中的签名文档重复问题。

---

## 🎯 功能新增

### 深度学习算法模块

新增 `pynetim.algorithms.deep_learning` 模块，集成基于深度强化学习的影响力最大化算法。

#### 支持的算法

| 算法 | 论文 | 会议/期刊 |
|------|------|-----------|
| ToupleGDD | ToupleGDD: A Fine-Designed Solution of Influence Maximization by Deep Reinforcement Learning | IEEE TCSS 2024 |
| S2V-DQN | Learning Combinatorial Optimization Algorithms over Graphs | NeurIPS 2017 |
| BiGDN | BiGDN: An End-to-End Influence Maximization Framework Based on Deep Reinforcement Learning and Graph Neural Networks | Expert Syst. Appl. 2025 |
| BiGDNS | BiGDN 学生模型，支持知识蒸馏 | - |

#### 核心组件

| 组件 | 说明 |
|------|------|
| `ToupleGDDAlgorithm` | 三重门控图神经网络 + DQN 算法 |
| `S2VDQNAlgorithm` | Structure2Vec + DQN 算法 |
| `BiGDNAlgorithm` | 端到端图神经网络 + DQN 算法（教师模型） |
| `BiGDNSAlgorithm` | BiGDN 学生模型，支持知识蒸馏 |
| `ToupleGDDTrainer` | ToupleGDD 训练器 |
| `S2VDQNTrainer` | S2V-DQN 训练器 |
| `BiGDNTrainer` | BiGDN/BiGDNS 训练器 |
| `BiGDNNodeEncoderTrainer` | BiGDN NodeEncoder 预训练器，用于 BiGDN 模型初始化 |

#### 使用示例

**推理模式**:
```python
from pynetim import IMGraph
from pynetim.algorithms import ToupleGDDAlgorithm, S2VDQNAlgorithm, BiGDNAlgorithm, BiGDNSAlgorithm

graph = IMGraph(edges, weights=0.3)

# ToupleGDD (默认 use_topk=True)
algo = ToupleGDDAlgorithm(graph, pretrained=True)
seeds = algo.run(k=10)  # 一次性选择 top-k
seeds = algo.run(k=10, use_topk=False)  # 迭代选择

# S2V-DQN (默认 use_topk=False)
algo2 = S2VDQNAlgorithm(graph, pretrained=True)
seeds = algo2.run(k=10)  # 迭代选择

# BiGDN
bigdn = BiGDNAlgorithm(graph, pretrained=True)
seeds = bigdn.run(k=10)

# BiGDNS (学生模型，需要教师模型)
bigdns = BiGDNSAlgorithm(graph, teacher_path='teacher.pth', pretrained=True)
seeds = bigdns.run(k=10)
```

**训练模式**:
```python
from pynetim.algorithms import (
    ToupleGDDTrainer, S2VDQNTrainer,
    BiGDNTrainer, BiGDNNodeEncoderTrainer
)

# ToupleGDD 训练
touplegdd_trainer = ToupleGDDTrainer(
    device='auto',
    double_dqn=True
)

touplegdd_trainer.train(
    graphs=[graph],
    budget=10,
    num_epochs=100,
    save_path='touplegdd.ckpt'
)

# S2V-DQN 训练
s2vdqn_trainer = S2VDQNTrainer(
    device='auto',
    double_dqn=True
)

s2vdqn_trainer.train(
    graphs=[graph],
    budget=10,
    num_epochs=100,
    save_path='s2vdqn.ckpt'
)

# BiGDN 训练（需要先预训练 NodeEncoder）
ne_trainer = BiGDNNodeEncoderTrainer(
    num_features=64,
    device='auto'
)
ne_trainer.train(
    graphs=train_graphs,
    num_epochs=100,
    save_path='encoder.pth'
)

bigdn_trainer = BiGDNTrainer(
    num_features=64,
    device='auto',
    encoder_path='encoder.pth'
)

bigdn_trainer.train(
    graphs=[graph],
    budget=10,
    num_epochs=100,
    save_path='bigdn_model.pth'
)

# BiGDNS 训练（学生模型，使用知识蒸馏）
bigdns_trainer = BiGDNTrainer(
    num_features=64,
    device='auto',
    is_student=True,
    teacher_path='teacher.pth'
)

bigdns_trainer.train(
    graphs=[graph],
    budget=10,
    num_epochs=100,
    save_path='bigdns_model.pth'
)
```

#### 训练特性

| 特性 | 说明 |
|------|------|
| n-step Q-learning | 支持多步奖励累积 |
| Double DQN | 减少过估计偏差 |
| epsilon-greedy | 平衡探索与利用 |
| 经验回放 | 提高样本效率 |
| 目标网络 | 稳定训练过程 |
| 知识蒸馏 | BiGDNS 支持从教师模型学习 |

#### NodeEncoder 预训练

BiGDN 算法需要预训练 NodeEncoder，用于学习节点特征表示：

```python
from pynetim.algorithms.deep_learning import BiGDNNodeEncoderTrainer

ne_trainer = BiGDNNodeEncoderTrainer(
    num_features=64,
    lr=0.001,
    device='auto'
)
encoder = ne_trainer.train(
    graphs=train_graphs,
    num_epochs=100,
    num_trials=10000,  # MC 模拟次数
    save_path='encoder.pth'
)
```

预训练原理：通过监督学习，预测每个节点单独作为种子时的归一化影响力值。

#### 奖励计算方法

| 方法 | 说明 |
|------|------|
| MC (蒙特卡洛) | 使用 C++ 传播模型多线程模拟 |
| RR (反向可达) | 使用 C++ RR 集合生成，快速准确 |

## 🐛 Bug 修复

### pybind 签名文档重复问题

算法模块的 pybind 绑定文件缺少 `disable_function_signatures()` 调用，导致 Python `help()` 文档中出现两个函数签名。

**问题示例**:
```
# 修复前 - 显示两个签名
__init__(...)
__init__(graph: IMGraph, model: str, ...) -> None
```

**修复文件**:
- `src/pynetim/cpp/bindings/algorithms/imm_bind.cpp`
- `src/pynetim/cpp/bindings/algorithms/tim_bind.cpp`
- `src/pynetim/cpp/bindings/algorithms/opim_bind.cpp`
- `src/pynetim/cpp/bindings/algorithms/base_ris_bind.cpp`

**修复方法**: 在类定义外包裹 `py::options options; options.disable_function_signatures();`

### pyi 存根文件参数不一致

#### OPIMAlgorithm.run() 多余参数

`opim_algorithm.pyi` 中 `OPIMAlgorithm.run()` 和 `OPIMCAlgorithm.run()` 方法包含 cpp 实现中不存在的 `mode` 参数。

**修复**: 移除 `mode` 参数

| 类 | 方法 | 修复前 | 修复后 |
|---|------|--------|--------|
| OPIMAlgorithm | run | `run(k, num_rr_sets, delta, mode)` | `run(k, num_rr_sets, delta)` |
| OPIMCAlgorithm | run | `run(k, epsilon, delta, mode)` | `run(k, epsilon, delta)` |

#### SI/SIR 模型缺少方法绑定

cpp 头文件中定义了 `set_beta()`, `set_max_steps()`, `set_gamma()` 方法，但绑定文件中未暴露。

**修复**: 添加方法绑定

| 模型 | 新增方法 |
|------|----------|
| SusceptibleInfectedModel | `set_beta()`, `set_max_steps()` |
| SusceptibleInfectedRecoveredModel | `set_beta()`, `set_gamma()` |

#### IMGraph 缺少方法

`graph.pyi` 缺少 `get_adj_list_py()` 方法的类型提示。

**修复**: 添加 `get_adj_list_py()` 方法声明

## ✨ 改进

### 友好错误提示

pybind11 默认的类型错误信息冗长且难以阅读，特别是当参数包含大量数据时（如长边列表）。

**问题示例**:
```
# 修复前 - 冗长的错误信息
__init__(): incompatible constructor arguments. The following argument types are supported:
    1. pynetim.graph.graph.IMGraph(edges: object, weights: object = 1.0, directed: bool = True, renumber: bool = True)

Invoked with: 1, [(0, 1), (1, 2), (2, 3), ...]; kwargs: weights=1.0
```

**修复后**:
```
TypeError: IMGraph() 参数错误: edges 必须是 list[tuple[int, int]] 类型。
用法: IMGraph(edges, weights=1.0, directed=True, renumber=True)
```

**修改范围**: 所有 pybind 绑定文件的构造函数参数验证

| 模块 | 验证参数 |
|------|----------|
| IMGraph | `edges`, `weights` |
| IndependentCascadeModel | `graph` |
| LinearThresholdModel | `graph` |
| SusceptibleInfectedModel | `graph` |
| SusceptibleInfectedRecoveredModel | `graph` |
| PyDiffusionModelBase | `graph` |
| IMMAlgorithm | `graph` |
| TIMAlgorithm | `graph` |
| OPIMAlgorithm | `graph` |
| BaseRISAlgorithm | `graph` |
