## [0.5.0] - 2026-04-05

### 重大变更 (Breaking Changes)
- **模块结构扁平化**:
  - 将 `src/pynetim/cpp/` 下的所有内容移动到 `src/pynetim/` 主目录
  - 删除 `cpp` 子目录，C++ 模块直接作为主模块
  - 新的导入方式：
    ```python
    # 之前 (v0.4.5)
    from pynetim.cpp.graph import IMGraphCpp
    from pynetim.cpp.diffusion_model import IndependentCascadeModel
    
    # 现在 (v0.5.0)
    from pynetim import IMGraph
    from pynetim import IndependentCascadeModel
    # 或
    from pynetim.graph import IMGraph
    from pynetim.diffusion_model import IndependentCascadeModel
    ```

- **`pynetim.py` 模块进入维护模式**:
  - 该模块不再添加新功能，仅进行维护和 bug 修复
  - 导入时会显示警告提醒用户使用主模块
  - 新功能和性能优化集中在 C++ 主模块 (`pynetim`)
  - 推荐迁移：
    ```python
    # 旧方式（已弃用）
    from pynetim.py.graph import IMGraphPy
    from pynetim.py.diffusion_model import IndependentCascadeModel
    
    # 新方式（推荐）
    from pynetim import IMGraph, IndependentCascadeModel
    from pynetim.diffusion_model import BaseCallbackDiffusionModel
    ```

- **`out_neighbors()` 返回类型变更**:
  - 之前：返回 `List[Edge]` 对象，需要通过 `edge.to` 和 `edge.weight` 访问
  - 现在：直接返回 `List[Tuple[int, float]]`，更简洁高效
  - 使用方式变更：
    ```python
    # 之前
    for edge in graph.out_neighbors(node):
        neighbor, weight = edge.to, edge.weight
    
    # 现在
    for neighbor, weight in graph.out_neighbors(node):
        ...
    ```

- **自定义传播模型模块位置变更**:
  - `PyDiffusionModelBase` 从 `pynetim.py.diffusion_model` 移至 `pynetim.diffusion_model`
  - 新增 `BaseCallbackDiffusionModel` 抽象基类供用户继承

### 新增 (Added)
- **自定义传播模型支持 (Python-C++ 混合架构)**:
  - 新增 `PyDiffusionModelBase` C++ 基类，允许 Python 用户继承并实现自定义传播逻辑
  - 新增 `BaseCallbackDiffusionModel` Python 抽象基类，位于 `pynetim.diffusion_model` 主模块
  - 用户只需实现 `run_single_trial(seeds, rng_seed)` 方法
  - 自动享受 C++ 层提供的蒙特卡洛模拟等基础设施
  - **注意**: 由于 Python GIL 限制，多线程模式下 Python 回调会串行执行，建议使用单线程模式以获得最佳性能
  - 性能：比纯 Python 实现快 1.5-3.5 倍
  - 使用示例：
    ```python
    from pynetim.diffusion_model import BaseCallbackDiffusionModel
    
    class MyICModel(BaseCallbackDiffusionModel):
        def run_single_trial(self, seeds, rng_seed):
            import random
            random.seed(rng_seed)
            activated = set(seeds)
            current = list(seeds)
            count = len(seeds)
            frequency = [0] * self.graph.num_nodes
            
            while current:
                new_active = []
                for node in current:
                    for neighbor, weight in self.graph.out_neighbors(node):
                        if neighbor not in activated and random.random() < weight:
                            activated.add(neighbor)
                            new_active.append(neighbor)
                            count += 1
                            frequency[neighbor] += 1
                current = new_active
            
            return count, activated, frequency
    
    model = MyICModel(graph, {0, 1})
    avg = model.run_monte_carlo_diffusion(1000)
    ```

- **多进程版传播模型支持**:
  - 新增 `BaseMultiprocessDiffusionModel` 抽象基类，支持真正的多进程并行
  - 解决 `BaseCallbackDiffusionModel` 的 GIL 限制问题
  - 使用 Python `multiprocessing` 实现真正的并行计算
  - **性能对比**（1000 节点，10000 边，2000 次模拟）：

    | 模型 | 并行度 | 耗时 | 说明 |
    |------|--------|------|------|
    | C++ IC | 1 线程 | 0.30s | 基准（最快） |
    | C++ IC | 4 线程 | 0.38s | 多线程加速 |
    | BaseCallbackDiffusionModel | 1 线程 | 6.6s | 单线程可用 |
    | BaseCallbackDiffusionModel | 4 线程 | 6.9s | ⚠️ GIL 限制，无加速 |
    | BaseMultiprocessDiffusionModel | 1 进程 | 6.2s | 单进程 |
    | BaseMultiprocessDiffusionModel | 4 进程 | 3.7s | ✅ 真正并行加速 |

  - **两种自定义模型对比**：

    | 特性 | `BaseCallbackDiffusionModel` | `BaseMultiprocessDiffusionModel` |
    |------|---------------------------|---------------------------|
    | 架构 | Python-C++ 混合（C++回调） | 纯 Python（多进程） |
    | 多线程 | ❌ GIL 限制无加速 | N/A |
    | 多进程 | N/A | ✅ 真正并行加速 |
    | 适用场景 | 单线程模拟 | 需要并行加速的自定义模型 |

  - 使用示例：
    ```python
    from pynetim.diffusion_model import BaseMultiprocessDiffusionModel
    
    class MyICModel(BaseMultiprocessDiffusionModel):
        def run_single_trial(self, seeds, rng_seed):
            # 同上实现
            ...
    
    model = MyICModel(graph, {0, 1})
    avg = model.run_monte_carlo_diffusion(1000, num_processes=4)
    ```

- **影响力最大化算法模块**:
  - 新增 `pynetim.algorithms` 模块，包含多种影响力最大化算法
  - **启发式算法**:
    - `SingleDiscountAlgorithm` - 简单度折扣算法
    - `DegreeDiscountAlgorithm` - 度折扣算法
  - **模拟类算法**:
    - `GreedyAlgorithm` - 贪婪算法
    - `CELFAlgorithm` - CELF 优化算法
    - `CELFPlusAlgorithm` - CELF++ 优化算法 (WWW 2011)
  - **RIS 类算法**:
    - `BaseRISAlgorithm` - 基础反向影响采样算法
    - `IMMAlgorithm` - IMM 算法 (SIGMOD 2015)
    - `TIMAlgorithm` - TIM 算法
    - `TIMPlusAlgorithm` - TIM+ 算法
  - **OPIM 类算法** (SIGMOD 2018):
    - `OPIMAlgorithm` - Online Processing for Influence Maximization，提供可证明的近似保证
    - `OPIMCAlgorithm` - OPIM-C 自适应采样版本，迭代达到 (1-1/e-ε) 近似保证

- **类型提示改进**:
  - 使用 `py::options::disable_function_signatures()` 禁用 pybind11 自动生成的复杂类型签名
  - `help()` 现在显示清晰的类型提示：
    - `int` 而不是 `typing.SupportsInt | typing.SupportsIndex`
    - `list[tuple[int, int]]` 而不是 `collections.abc.Sequence[...]`
    - `set[int]` 而不是 `collections.abc.Set[...]`
  - 所有 C++ 绑定模块的 docstring 都包含清晰的签名和参数说明

### 修复 (Fixed)
- **SIR/SI 模型参数验证**:
  - 移除 `beta` 和 `gamma` 参数的默认值，用户必须显式提供
  - 避免 `gamma=0.0` 与 C++ 层 `(0,1]` 范围要求的冲突
  - 使用示例：
    ```python
    # 之前（v0.4.x）- 有默认值，可能导致错误
    model = SusceptibleInfectedRecoveredModel(graph, seeds)  # gamma=0.0 会报错
    
    # 现在（v0.5.0）- 必须显式提供
    model = SusceptibleInfectedRecoveredModel(graph, seeds, beta=0.1, gamma=0.05)
    ```

- **`set_record_activation_frequency(True)` 段错误修复**:
  - 修复 `count_per_node` 向量未初始化大小导致的内存越界访问
  - 现在可以正常使用激活频数记录功能

- **图 API 改进**:
  - 新增 `has_edge(u, v)` 方法，检查边是否存在
  - `get_edge_weight(u, v)` 边不存在时抛出 `RuntimeError`（之前静默返回 0.0）
  - `batch_get_edge_weight(edges, default_value=0.0, raise_on_missing=False)` 新增参数：
    - `default_value`: 边不存在时的默认返回值
    - `raise_on_missing`: 边不存在时是否抛出异常
  - Python 版本 `IMGraphPy` 同步添加 `has_edge()` 和 `get_edge_weight()` 方法
  - 使用示例：
    ```python
    # 检查边是否存在
    if graph.has_edge(0, 1):
        weight = graph.get_edge_weight(0, 1)
    
    # 批量获取权重，边不存在时返回默认值
    weights = graph.batch_get_edge_weight([(0, 1), (1, 0)], default_value=-1)
    # 返回: [0.8, -1.0]  (假设边 (1, 0) 不存在)
    
    # 批量获取权重，边不存在时抛出异常
    weights = graph.batch_get_edge_weight([(0, 1), (1, 0)], raise_on_missing=True)
    # 抛出 RuntimeError: Edge (1, 0) does not exist
    ```

- **类型提示修复**:
  - 修复 `graph.pyi` 中 `__init__` 签名与实际 API 不符的问题
  - 添加缺失的 `out_neighbors_arrays` 方法声明
  - 添加 `original_to_internal` 和 `internal_to_original` 属性声明

- **性能优化**:
  - `to_scipy_sparse()` 使用 C++ `get_adj_matrix_sparse()` 替代 Python 循环
  - `to_pyg()` 使用 C++ `get_adj_matrix_sparse()` 替代 Python 循环
  - `to_networkx()` 使用 `batch_out_neighbors_with_weights()` 批量获取邻居
  - `to_igraph()` 使用 `batch_out_neighbors_with_weights()` 批量获取邻居
