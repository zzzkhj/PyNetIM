.. py:class:: BaseDiffusionModel(graph: pynetim.graph.IMGraph, init_seeds: list, record_states: bool = False)

   扩散模型基类。

   该类为各种扩散模型实现提供了基础框架，包含图结构、初始种子集、状态记录功能，
   以及需要子类实现的核心方法。支持在网络图上模拟信息、疾病或其他
   实体的传播过程。

   .. attribute:: graph

      表示传播网络结构

      :type: IMGraph

   .. attribute:: init_seeds

      初始种子集

      :type: list

   .. attribute:: record_states

      指示是否记录传播过程中的状态变化

      :type: bool

   .. attribute:: states

      当record_states为True时存储传播过程的状态历史

      :type: list


   .. py:attribute:: graph


   .. py:attribute:: init_seeds


   .. py:attribute:: record_states
      :value: False



   .. py:method:: update()
      :abstractmethod:


      更新模型状态的抽象方法。

      子类必须实现此方法来定义具体的更新逻辑。
      每次调用应执行一次传播步骤。

      :returns: 本轮更新结果，具体类型由子类定义

      :raises NotImplementedError: 当子类未实现此方法时抛出



   .. py:method:: diffusion(update_counts: int = None)
      :abstractmethod:


      执行完整扩散过程的抽象方法。

      子类必须实现此方法来定义从开始到结束的完整传播流程。

      :param update_counts: 更新次数，控制扩散迭代的轮数
      :type update_counts: int, optional

      :returns: 扩散结果，具体类型由子类定义

      :raises NotImplementedError: 当子类未实现此方法时抛出



   .. py:method:: run_monte_carlo_diffusion(round: int, multi_process: bool = False, processes: int = None)
      :abstractmethod:


      执行蒙特卡洛模拟扩散过程。

      :param round: 模拟的轮数
      :type round: int
      :param multi_process: 指示是否使用多进程进行模拟
      :type multi_process: bool
      :param processes: 多进程的进程数，默认为None，表示使用CPU核数
      :type processes: int, optional

      :returns: 模拟结果，具体类型由子类定义

      :raises NotImplementedError: 当子类未实现此方法时抛出



   .. py:method:: reset(init_seeds=None)

      重置模型状态。

      将模型恢复到初始状态，如果启用状态记录则清空已记录的状态历史，
      并重新设置初始种子节点状态。

      :param init_seeds: 新的初始种子节点集合，若为None则使用原有种子集
      :type init_seeds: list, optional
